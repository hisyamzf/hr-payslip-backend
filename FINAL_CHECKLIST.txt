📝 API ENDPOINTS

GET  /health
     Health check endpoint
     
GET  /
     Root info endpoint
     
GET  /docs
     Swagger UI documentation
     
GET  /openapi.json
     OpenAPI schema
     
GET  /api/v1/payslips/{payslip_id}/pdf
     Download or view payslip PDF
     Query params:
       - download=true  → Download as attachment
       - download=false → View inline in browser
     
GET  /api/v1/payslips/employee/{employee_id}/period/{period_start}
     Get payslip by employee and period
     Example: /api/v1/payslips/employee/uuid/period/2025-03-01
     
GET  /api/v1/payslips/employee/{employee_id}
     List all payslips for employee
     Query params:
       - limit  (default: 20, max: 100)
       - offset (default: 0)

🔄 INTEGRATION POINTS

UploadService.process_upload() FLOW:
  1. Parse Excel file
  2. Validate each row
  3. Transform to payslips
  4. Batch insert to database
  5. Call _queue_pdf_generation() ← NEW
  6. Return results

PDF Generation FLOW:
  1. Frontend requests: GET /api/v1/payslips/{id}/pdf
  2. API routes to payslips.py endpoint
  3. Fetch payslip from database
  4. Initialize PayslipPDFService
  5. Render Jinja2 HTML template
  6. Convert HTML → PDF with WeasyPrint
  7. Return PDF bytes (download or inline)

⚠️  IMPORTANT NOTES

1. WeasyPrint System Libraries Limitation
   - WeasyPrint requires system libraries (libgobject, pango)
   - These are NOT available on Windows natively
   - Solutions:
     a) Use WSL2 (Windows Subsystem for Linux 2)
     b) Use Docker container
     c) Deploy to Linux server
   - API still works without WeasyPrint (lazy-loaded)
   - PDF generation will fail gracefully with clear error

2. No Breaking Changes
   - All existing code remains unchanged
   - All existing functionality preserved
   - Only ADDITIONS, no modifications to core logic
   - Excel upload flow unchanged
   - Database schema unchanged

3. Lazy Loading Pattern
   - WeasyPrint imports only when PDF generation is needed
   - Server can start and run other endpoints without WeasyPrint
   - Graceful error handling if WeasyPrint unavailable
